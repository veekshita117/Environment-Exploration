# 🤖 Autonomous Environment Exploration

**Team Blacksheeps**

A ROS 2 package that autonomously explores an unknown environment using frontier-based exploration. The robot analyzes the occupancy grid map in real time, identifies unexplored frontiers, and navigates toward them until the entire reachable area has been mapped.

---

## 📖 Overview

This project implements a **frontier exploration algorithm** for a mobile robot in ROS 2. Instead of relying on a pre-built map or manual teleoperation, the robot:

1. Subscribes to the live occupancy grid (`/map`) and its own odometry (`/odom`)
2. Scans the map for **frontiers** — boundaries between known free space and unexplored space
3. Scores each frontier candidate based on **distance** and **information gain** (how much unknown area surrounds it)
4. Publishes the best frontier as a navigation goal (`/goal_pose`) for the navigation stack to drive toward
5. Repeats until no frontiers remain, at which point exploration is marked complete

A full project report and a demo video (RViz visualization) are included in this repo.

---

## ✨ Features

- 🗺️ Real-time frontier detection from `OccupancyGrid` data
- 📍 Goal scoring using distance + local information gain (favors frontiers that reveal more unknown space)
- 🚫 Duplicate-goal avoidance with a minimum distance threshold and a cooldown timer
- 🔁 Fully autonomous loop — no manual driving required
- 🧩 Built as a standard ROS 2 Python (`ament_python`) package

## 🛠️ Requirements

- ROS 2 (developed and tested on **Humble**; should work on other recent distros)
- Python 3
- ROS 2 packages: `rclpy`, `nav_msgs`, `geometry_msgs`, `sensor_msgs`
- A SLAM stack (e.g. `slam_toolbox`) publishing `/map`
- A navigation stack (e.g. Nav2) subscribed to `/goal_pose`

## 🚀 Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/<your-username>/Team_Blacksheeps.git
cd Team_Blacksheeps/exploration_ws
```

### 2. Build the workspace

```bash
colcon build
source install/setup.bash
```

### 3. Run the explorer node

Make sure your robot's SLAM node (publishing `/map` and `/odom`) and navigation stack (subscribed to `/goal_pose`) are already running, then:

```bash
ros2 run autonomous_explorer explorer
```

The robot will begin identifying frontiers and publishing navigation goals automatically. Exploration is considered complete once no frontiers remain in the map.

## 🧠 How the Algorithm Works

The core logic lives in [`explorer.py`](./exploration_ws/src/autonomous_explorer/autonomous_explorer/explorer.py):

- **Frontier detection:** scans every free cell in the occupancy grid and flags it as a frontier if any of its 4-connected neighbors is unknown (`-1`)
- **Scoring:** each frontier's score = `distance_from_robot - (0.02 × info_gain)`, where `info_gain` counts unknown cells in a 7×7 window around the candidate — lower score wins
- **Goal filtering:** frontiers closer than `0.8 m` to the robot are skipped to avoid redundant goals
- **Goal locking:** once a goal is published, new goals are paused for `18 s` (or until the robot reaches it) to give the navigation stack time to act
